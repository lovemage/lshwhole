# DOSO Import Checkpoint & Skip Existing Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add resumable DOSO import sessions with checkpointing, auto-skip for previously imported products, single-target URL input, and visible total/progress counters in admin UI.

**Architecture:** Introduce two DB-backed tables (`doso_import_sessions`, `doso_import_items`) as the source of truth for progress and dedup status. Split current import API into session-based endpoints (`start`, `run`, `progress`, `pause`) and refactor DOSO service functions to process pending items in batches. Update admin UI to operate by session state and render real-time progress metrics.

**Tech Stack:** Next.js App Router, TypeScript, Supabase (Postgres), existing Playwright-based DOSO probe/import service, ESLint, `tsc --noEmit`.

**Step 3: Write minimal migration**

```sql
create table if not exists doso_import_sessions (
  id bigint generated by default as identity primary key,
  admin_user_id uuid,
  target_url text not null,
  status text not null default 'pending',
  total_count integer not null default 0,
  processed_count integer not null default 0,
  imported_count integer not null default 0,
  skipped_count integer not null default 0,
  failed_count integer not null default 0,
  last_checkpoint_product_code text,
  error_message text,
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);
```

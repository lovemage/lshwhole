### 使用流程（營運）

1. 後台輸入 DOSO 帳密，先按「同步來源分類」。
2. 設定 L1 日本、來源分類映射與目錄 fallback，按「儲存映射」。
3. 執行 DOSO 導入（同步商品 / 導入網站）。
4. 單筆或批量上架時，系統會自動帶入分類。

### 失敗排查

- 出現 `missing_category_mapping`：
  - 先檢查該商品是否帶有 `source_category_id` 與目錄資訊
  - 再檢查該來源分類映射是否已設定
  - 若來源分類缺失，確認該目錄 fallback 是否有設定

# DOSO Probe MVP 說明

## 目標

- 在管理員後台直接輸入 DOSO 帳密進行目錄探測。
- 探測僅回傳報告，不寫入商品資料庫。
- 不保存帳密（一次性輸入、請求結束即丟棄）。

## 目前 Probe 範圍

- 支援目錄：
  - `https://www.doso.net/onlineMall/selfOperatedMall`
  - `https://www.doso.net/onlineMall/PreSelfOperatedMall`
  - `https://www.doso.net/onlineMall/etonet`
  - `https://www.doso.net/onlineMall/etonetRanking`
  - `https://www.doso.net/onlineMall/tanbaya`
  - `https://www.doso.net/onlineMall/dabandaxi`
  - `https://www.doso.net/onlineMall/dabansinei`
  - `https://www.doso.net/onlineMall/shineiRanking`
  - `https://www.doso.net/onlineMall/gomen`

## 操作流程

1. 進入管理後台 → 爬蟲導入頁。
2. 輸入 DOSO 帳號與密碼。
3. 可調整目錄 URL 清單（空白或換行分隔）。
4. 點擊「探測目錄」。
5. 查看每個目錄的 `list_ok`、`list_count_page`、`detail_ok` 與樣本資料。

## 安全設計

- API 不回傳密碼。
- 前端送出後會清空密碼欄位。
- 探測完成即關閉 browser context，不保留登入狀態。

# DOSO 帳密儲存（加密）

## 功能摘要

- 管理後台可儲存一組共用 DOSO 帳密。
- 密碼以 AES-256-GCM 加密後儲存於 `system_settings`，前端不回傳明文密碼。
- 導入/探測時，若本次未輸入帳密，會自動使用已儲存帳密。

## API

- `GET /api/admin/sync/doso/credentials`
- `PUT /api/admin/sync/doso/credentials`

## 環境變數

- `DOSO_CREDENTIALS_ENCRYPTION_KEY`（必填）

說明:

- 建議使用 32-byte key（可用 base64 或 hex）。
- 若未設定或格式錯誤，儲存帳密 API 會回錯誤。

## UI

- DOSO 導入區塊新增「儲存帳密」按鈕。
- 標題旁新增「使用方式」連結，點擊後顯示操作彈窗。
